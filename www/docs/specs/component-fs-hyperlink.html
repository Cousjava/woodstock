1.2 Component Description

The hyperlink JSF component now renders itself with the help of a client side renderer instead of an html renderer. This means that there will exist a client side hyperlink widget for rendering the hyperlink component.

1.3 Component Purpose

The Hyperlink component does not introduce any new functionality or interfaces. The com.sun.webui.jsf.renderkit.widget.HyperlinkRenderer renders the Hyperlink component as a javascript dojo widget which allows developer to control its behavior on the client.

3.0 Component Usage

The hyperlink component can be used in a JSP tag the following way.
     
 <webuijsf:hyperlink  id="hyperlinktest2"  text="Click me" actionExpression="#{HyperlinkBean.success}"/>

Using the JSP tag, all the in-line JavaScript necessary to generate a Dojo widget is automatically output in the rendered HTML page. This includes the name of the desired widget, any localized and themed properties required to populate the HTML template, and any JavaScript files (e.g., widget JavaScript) necessary to support the Dojo Widget.

  Note that either the actionExpression or the url attribute can be used for page navigation. More information on how the hyperlink works with the url and the actionExpression
  attributes specified can be found at section 8.0

  For the hyperlink component that was created using the above mentioned custom JSP tag, the inline javascript that is output will look like: 
dojo.require('webui.<theme>.widget.*');
webui.<theme>.widget.common.createWidget({
    "contents": ["Click me"],
    "formId": "form1",
    "templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/anchor.html",
    "_widgetType": "webui.<theme>:hyperlink",
    "params": [],
    "disabled": false,
    "visible": true,
    "_modules": ["webui.<theme>.widget.hyperlink"],
    "id": "form1:hyperlinktest2"
});


The hyperlink widget can be updated in the following way:
    • Get an handle to to the hyperlink dom node
      var domNode =  document.getElementById("form1:link1")
    • Use the getProps function that is available on the dom node to obtain all the properties that are set on that particular dom element.
      •  var props = domNode.getProps(); // returns a json object.
    • Use the setProps function to specify the properties to be updated for the particular component as a json object.
       domNode.setProps({disabled:true});
Note: <appname> is replaced by the application name.
         <theme> tag is replaced by the theme name.

3.1 Updating the component Asynchronously

The hyperlink component also supports asynchronous updates. For example you could update the text of the component asynchronously the following way.
This example uses a text field whose value is bound to a backing bean. The hyperlink component's text value is also bound to the same backing bean.
When an user enters text in the text field, the corresponding text will be updated in the hyperlink.

Here is how the code looks like.

                    <webuijsf:hyperlink id="hyperlink7"  text="#{HyperlinkBean.dynamicText}" url="http://google.com"/>
                    <br/><br/>                           
                    <webuijsf:textField id="textField1" onKeyUp="refreshHyperlink();" text="#{HyperlinkBean.dynamicText}"
                      label="Enter new text"/><br/>  

Note that the textField's onKeyUp attribute refers the refreshHyperlink javascript function.
This function will invoke the "refresh" function that is located on the html hyperlink element's dom.
The argument that is passed to the refresh function would be the client id of the text field.
Hence the javascript function would look something like this.

                <script type="text/javascript">
                    function refreshHyperlink() {
                        // Get text to refresh.
                        var domNode = document.getElementById("form1:hyperlink7");
                        domNode.refresh("form1:textField1"); // Update text field value and refresh.
                    }
               </script>

By default, the hyperlink component's ajax functionality is set to true. i.e. the ajaxify attribute of the hyperlink component returns true by default.
You can turn off the ajax functionality by setting the ajaxify attribute to false while creating the hyperlink component.

                <webuijsf:hyperlink id="hyperlink7"  ajaxify="false" text="#{HyperlinkBean.dynamicText}" url="http://google.com"/>

The above created hyperlink component will not exhibit any ajax functionality.

4.7 Component Extensibility

Components which want to exhibit the functionality of a basic html anchor element should inherit from the anchor component rather than from an hyperlink component. The hyperlink widget outputs properties which help in form submission when the hyperlink is clicked and these may not be required for components whihc want to use an anchor element for just page navigations.

8.0 Component High level design

8.1 Client side widget


The hyperlink widget supports two different states called the enabled and disabled states. In the enabled state the hyperlink can be clicked and you can navigate to a different page. In the disabled state the hyperlink will appear as plain text. Clicking on the hyperlink doesnt generate any form of action in the disabled state. The hyperlink widget decides to whether submit the page or traverse to a specified url depending on the existence of the href attribtue set on the widget.
Note that the client side widget always defaults the dom element's href attribute to a value of "#". This is to help achieve the link look and feel. If an html anchor element does not have the href property set, then it renders as plain text. Hence, even though when the href property is not specified on the widget, the dom node's href attribute will default to "#". Hence, if a value of href is set on the widget (including the value "#"), then the hyperlink widget will traverse to the specified href value. Otherwise, it will submit the form.

8.2 JSF Renderer

Hyperlink's widget renderer has been done differently from its html renderer counterpart. The Hyperlink JSF component continues to inherit from the WebuiCommand class. The Hyperlink client side renderer nherits from the AnchorRendererBase. The AnchorRendererBase acts as a base renderer which outputs properties of an html  anchor element. The HyperlinkRenderer adds on to this functionality provided by this renderer and outputs properties that are specific to the Hyperlink.
