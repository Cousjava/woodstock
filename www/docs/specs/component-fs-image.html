1.2 Component Description

An image component is used to display an image on the page.

1.3 Component Purpose

The client side Image  does not introduce any new interfaces. The com.sun.webui.jsf.renderkit.widget.ImageRenderer renders the Image component as a javascript dojo widget which allows developer to control its behavior on the client.

1.5 Definition of Terms

  • HTML template - The file used to define HTML tags associated with a component
  • In-line JavaScript - JavaScript used to instantiate a Dojo Widget, located anywhere (i.e., in-line) in the rendered HTML page
  • JSF - JavaServer Faces
  • JSF component - The JSF UIComponent compliment to a JSP tag and JSF renderer
  • JSF facet - A facet is a subcomponent that has a specific purpose defined by its parent container
  • JSF renderer - The JSF Renderer compliment to a JSP tag and JSF component
  • JSON - JavaScript Object Notation (i.e., object literals) defined using braces ({}) that contain a set of comma separated key value pairs
  • JSP tag - The JSF UIComponentELTag or UIComponentELBodyTag compliment to a JSF component and JSF renderer
  • Subcomponent - A child component used to render a complex component
  • Widget - Entity consisting of widget JavaScript and CSS, not necessarily based on the Dojo toolkit
  • Widget JavaScript - The JavaScript used to populate an HTML template and/or update component properties
  • Contents - JSON Array which contain JSON objects of sub components to be displayed inside the widget
  • Widget renderer - The JSF renderer used to output in-line JavaScript for instantiating a Dojo Widget

3.0 Component Usage

The image component can be used to render an image specified by the url attribute or an image identified by a Theme identifier.
If  the url attribute is used, the jsp tag will look like this:

             <webuijsf:image id="image1"
          url="/images/version_product_name.png"
          alt="#{msgs.masthead_imageAltText}"
                        height="140" width="140" border="2" hspace="5" vspace="5" align="middle"
                        toolTip="#{msgs.masthead_imageToolTip}"
                        onMouseOver="(document.getElementById('form6:text')).innerHTML='#{msgs.masthead_imageMouseOver}'; return false;"
                        onMouseOut="(document.getElementById('form6:text')).innerHTML='#{msgs.masthead_imageMouseOut}'; return false;"
                        onClick="(document.getElementById('form6:text')).innerHTML='#{msgs.masthead_imageOnClick}'; return false;"/> 

The output json in this case would look something like this:

<script type="text/javascript">
webui.<theme>.widget.common.createWidget({
    "_widgetType": "webui.<theme>:image",
    "templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/image.html",
    "width": 140,
    "title": "This is an image",
    "src": "/<appname>/images/version_product_name.png",
    "onMouseOut": "(document.getElementById('form6:text')).innerHTML='The mouse is outside the image'; return false;",
    "alt": "Image",
    "id": "form6:image1",
    "onMouseOver": "(document.getElementById('form6:text')).innerHTML='The mouse is over the image'; return false;",
    "height": 140,
    "align": "middle",
    "hspace": 5,
    "vspace": 5,
    "visible": true,
    "border": 2,
    "onClick": "(document.getElementById('form6:text')).innerHTML='The image has been clicked'; return false;",
    "_modules": [
        "webui.<theme>.widget.image",
        "webui.<theme>.widget.jsfx.image"
    ]
});
</script>

And if an icon attribute is used, the jsp tag will look like this:

              <webuijsf:image id="image2" icon="ALARM_CRITICAL_SMALL" hspace="5" longDesc="#{msgs.masthead_imageDesc}"/>

In this case, the widget renderer will take care of mapping the icon's image to an appropriate url and output that in the json properties.
Hence, the json output would look something like this.

<script type="text/javascript">
webui.<theme>.widget.common.createWidget({
    "_widgetType": "webui.<theme>:image",
    "templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/image.html",
    "width": 10,
    "longDesc": "This is an example of an image icon",
    "height": 10,
    "hspace": 5,
    "visible": true,
    "border": 0,
    "src": "/<appname>/theme/com/sun/webui/jsf/<theme>/images/alarms/critical_small.gif",
    "_modules": [
        "webui.<theme>.widget.image",
        "webui.<theme>.widget.jsfx.image"
    ],
    "id": "form6:image2"
});
</script>

The image widget can be updated in the following way:
  • Get an handle to to the image dom node.
    var domNode =  document.getElementById("form6:image1")
  • Use the getProps function that is available on the dom node to obtain all the properties that are set on that particular dom element.
    •  var props = domNode.getProps(); // returns a json object.
  • Use the setProps function to specify the properties to be updated for the particular component as a json object.
     domNode.setProps({src:"/images/foo.jpg"});

Note: <appname> is replaced by the application name.
         <theme> tag is replaced by the theme name.

3.1 Updating the component Asynchronously

The image component also supports asynchronous updates. For example you could update the src of the component asynchronously the following way.
This example uses a text field whose value is bound to a backing bean. The image component's src value is also bound to the same backing bean.
When an user enters text in the text field, the corresponding text will be updated in the anchor.

Here is how the code looks like.

              <webuijsf:image id="image6" url="#{ImageBean.imageUrl}" alt="#{msgs.masthead_imageAltText}"
                        toolTip="#{msgs.masthead_imageToolTip}"/>             
              <br/>                          
              <webuijsf:textField id="textField1" onKeyUp="refreshImage();" text="#{ImageBean.imageUrl}"
                label="#{msgs.masthead_textField1}"/><br/>

Note that the textField's onKeyUp attribute refers the refreshImage javascript function.
This function will invoke the "refresh" function that is located on the html image element's dom.
The argument that is passed to the refresh function would be the client id of the text field.
Hence the javascript function would look something like this.

                <script type="text/javascript">
                    function refreshImage() {
                        // Get text to refresh.
                        var domNode = document.getElementById("form1:image6");
                        domNode.refresh("form1:textField1"); // Update text field value and refresh.
                    }
               </script>

By default, the image component's ajax functionality is set to true. i.e. the ajaxify attribute of the image component returns true by default.
You can turn off the ajax functionality by setting the ajaxify attribute to false while creating the image component.

                        <webuijsf:image id="image6" url="#{ImageBean.imageUrl}" alt="#{msgs.masthead_imageAltText}" ajaxify="false"
                        toolTip="#{msgs.masthead_imageToolTip}"/>     

The above created image component will not exhibit any ajax functionality.

4.6.2 Image resources


These image resources are found in com/sun/webui/jsf/<theme>/images directory.
The identifiers for these images are defined in the com/sun/webui/jsf/<theme>/properties/images.properties file.

Each image identifer has a corresponding identifiers with suffix
"_WIDTH" which contains a numeric value indicating the width of the specified image.
"_HEIGHT" which contains a numeric value indicating the height of the specified image.
"_ALT" which contains the alt text to be shown for the image.


For example, for the theme image key "LABEL_INVALID_ICON", the following entries will be found in the images.properties file.

LABEL_INVALID_ICON=/com/sun/webui/jsf/suntheme/images/alerts/error_small.gif
LABEL_INVALID_ICON_HEIGHT=14
LABEL_INVALID_ICON_WIDTH=11
LABEL_INVALID_ICON_ALT=Label.invalidAltText

Here the value for the LABEL_INVALID_ICON_ALT key points to a key found in the com/sun/webui/jsf/<theme>/properties/messages.properties file.

These identifiers can be used as the value of "icon" attribute for the image component.

8.0 Component High level design

The Icon JSF component extends from the ImageComponent and is used for specifying images obtained from the theme. Both the Icon and the ImageComponent use the ImageRenderer for rendering the images. The Image's widget renderer does not distinguish rendering of components depending on whether the component is a ImageComponent or an Icon component. It first checks whether the icon attribute for the component has been specified and if so whether the url attribute has not been specified. If that is the case, then the ImageRenderer renders the icon represented by the icon attribute.
If the url attribute is specified, then the image specified by the url attribute is rendered. Note that over here, if both the icon and the url attribute are specified, then the image specified by the url attribute
will take precedence. Also, if both icon and url values are specified, then the values of width, height and alt for that particular themed image specified by the icon attribute will not be applicable to the image specified by the url attribute.

Internet Explorer browsers  with versions below 7.0 have problems displaying png images. Hence,  a special image transformation function is applied applied to these images. This function can be found in the messages.properties file with the key "Image.IEPngCSSStyleQuirk".For png images on IE browsers with version below 7.0, if  the height and width values are not specified, then the renderer outputs a default value of "100px" for both width and height.
