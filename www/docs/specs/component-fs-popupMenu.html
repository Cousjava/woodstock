1.2 Component Description

The popup menu component displays a menu which contains a list of actions that can be invoked by clicking on it.

1.3 Component Purpose

The component is intended to be used in cases where there can be more than one possible action related to a particular event of a component (such as clicking an hyperlink.) In such cases, invoking the particular event on the component will open the popup menu which shows the list of possible actions that can be performed for that particular event.

2.1.2 Functional requirements

  • The popup menu component shall be able to associate itself with any component that can generate events.
  • The popup menu shall show a list of available actions associated with the particular event.
  • It should be possible to update the contents of the menu client side.
  • The options available in the menu when clicked can either generate an asynchronous request or submit the form.
  • The options present for each action can be either in an enabled state or in a disabled state.
  • The options available for a menu can be grouped based on some common functionalities.

3.0 Component Usage

3.1 Creating the popup menu

The <webuijsf:menu> custom JSP tag shall be used to create a  menu. This menu will contain options which can be grouped together based on some common functionality. The   menu uses the "items" attribute to specify the available options. The value specified for the options attribute can be either an array, Map or Collection whose members are all subclasses of the com.sun.webui.jsf.Option class.

Here is an example of how the popup menu can be created.

        <webuijsf:menu id="popup1" items="#{popUp.optionItems}" selected="#{popUp.selectedItem}"   />

Where the input to the options attribute could look something like this:
public class PopUp {

    private Option OptionItems[];
    private Option groupOptions[];
    private OptionGroup group;
    private String text="initial text";
    public PopUp() {
        OptionItems = new Option[5];
        OptionItems[0] = new Option("One", "one");
        OptionItems[1] = new Option("Two", "two");
        OptionItems[2] = new Option("Three", "three");
        OptionItems[4] = new Option("Five", "five");
        OptionItems[1].setImage("/images/dice1.gif");
        groupOptions = new Option[2];
        groupOptions[0] = new Option("SubOpt1", "SubOpt1");
        groupOptions[1] = new Option("SubOpt2", "SubOpt2");
       
        group = new OptionGroup();
        group.setValue("Group1");
        group.setLabel("Group1");
        group.setOptions(groupOptions);
        OptionItems[3] = group;
       
    }
   
    public Option[] getOptionItems() {
        return this.OptionItems;
    }

    public void setOptionItems(Option[] OptionItems) {
        this.OptionItems = OptionItems;
    }
   
   
    public String getSelectedItem() {
        return this.text;
    }
   
    public void setSelectedItem(String text) {
        this.text = text;
    }

3.2 Invoking the popup menu

The widgets created using the above mentioned custom JSP tags are not related to any other widgets present in the page.What this means is that, any widget present
on the page can invoke them and make use of them. When the widget is created in the html page, its style is set to hidden by default. Hence, none of the popup
menu widgets that are present in the page are visible when the page is first rendered. If a widget present in the form wants to show a pop-up menu, it can invoke the
"open" method of the popupMenu widget.

For example, an hyperlink component can display the above created popup menu component when it is clicked as follows.

        <webuijsf:hyperlink text="Click here to open menu" onMouseDown="displayMenu(event);return false;" onClick="return false;" actionExpression="hyperlinkPage"/>     
<script type="text/javascript">
        function displayMenu(event) {
            document.getElementById("form1:popup1").open(event);
        }
</script>

The popup menu will close by
-  Selecting one of the options present in the menu
- Clicking outside the menu anywhere on the page
- Opening another menu on the page.

The popupMenu widget exposes a function called "getSelectedOption" which will return the value of the selected option.

Note: <appname> - the name of the web application name.
         <theme> - The name of the theme used.

3.3 Submitting an selected option to the server.

When an option is selected on the popupMenu, then it sets a value on the widget which can be accessed by invoking domElement.getSelectedOption.
This gives the value of the selected element on the client side.The popupMenu does  not submit the selected value to the server by default.
The application can inform the server of the selected value either by

 i) Use domNode.submit() :- The popupMenu element has a submit() function set on its dom. Invoking this function
    will cause the popupMenu widget to send an asynchronous request to the server and cause the popupMenu component
    to update itself. The popupMenu widget in this case, appends an attribute called value to the dynamic faces
    request made. The com.sun.webui.jsf.renderkit.ajax.PopupMenuRenderer will in this case look for 
    this particular value and update itself accordingly.

	ex:
        <webuijsf:popupMenu id="popup1" items="#{popUp.optionItems}" selected="#{popUp.selectedItem}" 
		 onChange="setTimeout('changeEvent();', 0);"/>
	<script type="text/javascript">
        function changeEvent() {
            document.getElementById("form1:popup1").submit();
        }
	</script>

 ii) Set the submitForm attribute to true:- This will cause the popupMenu component to submit the form whenever an
    option is selected in the menu. All the elements in the form will be submitted and all the components present
    in the form will go through the normal JSF processing lifecycle. 
    The popupMenu widget in this case will submit the value of the selected option similar to the way the hyperlink widget
    submits the form. It will append the selected value as a request parameter with the key being the client id of 
    the popupMenu appended with a "_submittedValue" suffix.
	
	ex:

        <webuijsf:menu id="popup1" immediate="true" items="#{popUp.optionItems}" selected="#{popUp.selectedItem}"  submitForm="true"/>


3.4 Accessing the selected option on the server

The menu component uses its own event mechanism to publish selected values  for the menu. This is similar to the "ActionSource" components
which publish events for other components to listen for. There exists an eventListenerExpression similar to your actionListenerExpression which
can be bound to a method which takes an ValueEvent as an argument. This ValueEvent object contains the menu's selected value.

If you want to navigate to a different page using this particular value, then use the eventExpression (also similar to the actionExpression)
attribute for defining navigation rules.

 You can also set the "Phase Id" during which the particular event should be broadcasted.
Use the "phaseId" attribute for doing this and pass on a valid Phase Identifier as an argument for this attribute.

Here is a code snippet as to how a text field would update its value when a menu submits a selected value (through asynchronous refresh/form submit)

    /**
     * EventListener which goes through the selected value and updates
     * the text field with the selected option.
     *
     * @param ve The ValueEvent
     */
    public void selectOption(com.sun.webui.jsf.event.ValueEvent ve) {
	// Get the selected outcome from the ValueEvent.
        String outcome = ve.getSelectedOption().toString();
        matchOption(outcome, getOptionItems());
    }
    
    /**
     * Match the outcome of the selected item to the matching option.
     * set the textfield's value to this option's label.
     * @param outcome The selected menu item.
     * @param options The options in the menu
     */
    private void matchOption(String outcome, Option[] options) {
        for (int i=0; i<options.length; i++) {
          if (options[i].getValue().equals(outcome)) {            
              this.text = options[i].getLabel();
              break;
          }
          if (options[i] instanceof OptionGroup) {
              matchOption(outcome, ((OptionGroup)options[i]).getOptions());
          }
          
        }              
    }

5.0 - Creator Design Time Support

5.1 Design-time Behavior

         A default menu has to be created each time a menu component is dragged and dropped on to the designer surface. Using the items attribute, a developer shall be able to bind the options to the menu using the visual value binding tool. The default menu that is created when the menu is initially dragged and dropped on to the surface shall contain a dummy list of options such as "item1, item2, item3" etc.. After the options has been bound, the menu's list of options shall change to "abc abc ..." to show that there has been some data entered for the menu. This is similar to the behavior of list family of items. The visible attribute of the menu shall be set to true during design time to enable the developer to configure the menu.

7.0 - A11Y, I18n, Compliance, Debugging

7.1 Accessibility (A11Y)

The menu shall be accessible without the help of a mouse. The user shall be able to open the menu using the "shift+F10" key sequence. Opening this menu
causes the default focus to be set on the first option item in the menu. Using the up and down arrow keys will enable the user to traverse through the
various menu options. Pressing the enter key on a particular menu option shall cause that menu option to be selected and the menu will close. The focus will
be on the item that invoked the menu. Pressing "Esc" key also will cause the menu to close and focus to return on the invokee.

8.0 Component High level design

        8.1 JSF Component class

The Menu component uses the EventSource interface for delivering the selected value of the menu to the application. The EventSource interface defines a minimal component interface to provide an application with simple GUI events. These GUI events reflect "gestures" made by the user, such as selecting a menu option, or any action that results in a reqeust value representing that action submitted in a request either by  "form.submit" or an XmlHttpRequest.
The MenuRenderer queues an ValueEvent if the submitted value matches one found in the option items.The ValueEvent object which inherits from the FacesEvent class contains the selected value of the menu component. This object will be passed on as an argument to the method bound to EventListenerExpression attribute. To obtain the selected value, the developer needs to have a method binding to this attribute and extract the value from this value binding. The EventExpression interface evaluates the method binding assigned to it and performs the page navigation if needed by invoking the NavigationHandler.
A PhaseId can also be set for the menu component which will instruct when the menu should actually broadcast the value event. This will be useful in cases where the menu's value should be processed irrespective of state of other components, such as an immediate required text field.

8.2 JSF Renderer class

Like other components, there are two renderers, one for normal requests and one for asynchronous requests. Unlike other components, the way the decode method functions
differs depending on the way the request is received by the component. If its a normal request ( a full fledged http request), then the client side widget will have appended
the selected option to the request parameter. Hence, the decode method will take the value out of the request parameter map and apply it as the submittedValue of the component.

On the other hand, if the request is an ajax request, then the value will be included in the ajax request sent by the widget. Hence the decode method will now look for the
"xjson" property which is a JSON object in the request parameter map. It will then read out the value corresponding to the "value" key in the JSON object and update the
component correspondingly.
