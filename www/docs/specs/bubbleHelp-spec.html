1.2 Component Description

Bubble component is a kind of  pop up window which appears on screen in response to certain mouse interaction to a particular screen element. 

1.3 Component Purpose

Purpose of this component is to provide detailed information to the end user when a mouse hover on a particular screen element.
The Bubble component can contain any application specified content. But shouldn't be a complex component like "AddRemove". Component usage guidelines will describe the possible use cases.

2.0 Requirements, goals, and assumptions

2.1 Requirements

UI and Functional requirements are listed separately.

2.1.1 UI requirements

1. Structure: Bubble should contain following structure elements:

Bubble Title : The title appears in title bar area.
Bubble Remove Control: Appears to the right of the bubble title bar. Closes the bubble window.
Bubble Content Area: The main text of the bubble.

2. The size of the bubble window will stay smaller than the main window that it appears over. The bubble window area should not exceed half the area of the main window.

3. Component Positioning: Component should be positioned such that is is fully rendered within the browser's boundary.

2.1.2 Functional requirements

1. Bubble Content: The pop up will contain a header and a body. The header will not be optional.
    The header will have title text and controls. The text can also be a html link and link will appear different
    from the text and the title.
2. At any given time only one bubble should be visible on screen.
3. Clicking on close button in header should close bubble.
4. Developer should be able to control "when to close" behavior of bubble.
5. Bubble window should not follow the mouse.
6. Developer should be able to put text, images and any component for bubble.

3.0 Component Usage

To include an bubble component in the JSP page, the web app developer will include a tag like this.

<webuijsf:bubble id="bubble" title="Bubble Help Title">
    <webuijsf:staticText text="Add help Text" />
</webuijsf:bubble>

Using the JSP tag, all the in-line JavaScript necessary to generate a Dojo widget is automatically output in the rendered HTML page. This includes the name of the desired widget, any localized and themed properties required to populate the HTML template, and any JavaScript files (e.g., widget JavaScript) necessary to support the Dojo Widget. However, a Dojo Widget can also be represented by the following JavaScript. This JavaScript makes the Woodstock component available to web app developers who cannot, or do not want to, use the JSP and/or JSF frameworks.

Note that any one of the key-value pairs of properties below can contain more key-value pairs, possibly used by subcomponents.

dojo.require("webui.<theme>.widget.bubble");
webui.<theme>.widget.common.createWidget({
    _widgetType: "webui.<theme>:bubble",
    templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/bubble.html",
    title "Bubble Help Title",
    id: "form1:bubble"
   
});

How to call BubbleHelp on components: Bubble component exposes two javascript api "open()" and "close()". "open()" accept "event" as argument. These two functions needs to be invoked by the component who wants a bubble for the component. "How long a bubble should stay and when to close" behavior can be controlled by developer.
Example: Button uses "onMouseOver" event to open a bubbleHelp and closes it on "onMouseOut" event.
<webuijsf:button onMouseOver="document.getElementById('form1:bubble').open(event);"
                     onMouseOut="document.getElementById('form1:bubble').close();"
                     primary="true"  id="btn3" text="Change Types" onClick="return changeTypes();"/>

When the component is rendered, a DOM object corresponding to the component is created. To manipulate the component on the client side, you may invoke functions on the DOM object. With reference to the DOM id, to change the helpTitle, invoke document.getElementById(id).setProps({title: "New Help Title"}).

8.0 Component High level design

8.1 Client side widget

The client side bubble widget makes use of the fillInTemplate function to generate a template based widget. The bubble widget first registers the setProps function on its widget in the fillInTemplate function and then makes a call to setProps function which sets the properties common to the bubble widget. A getProps() is also registered in the fillInTemplate which is used to retrieve attribute values which setProps() function uses.

BubbleHelp component exposes two api which needs to be called by the component which wants bubble window to open/close on a particular event.

8.2 JSF Renderer

Bubble renderer out puts all the required properties which is used by bubble widget to render component.

8.3 JSF Renderer (Ajax)

Bubble renderer (ajax) handles ashyncronous requests. This enables asynchronously refreshing the component
