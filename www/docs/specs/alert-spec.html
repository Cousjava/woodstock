The component still maintains its original purpose However the new client side rendering will enable updating alert component client side.

1.5 Definition of Terms

  • HTML template - The file used to define HTML tags associated with a component
  • In-line JavaScript - JavaScript used to instantiate a Dojo Widget, located anywhere (i.e., in-line) in the rendered HTML page
  • JSF - JavaServer Faces
  • JSF component - The JSF UIComponent compliment to a JSP tag and JSF renderer
  • JSF facet - A facet is a subcomponent that has a specific purpose defined by its parent container
  • JSF renderer - The JSF Renderer compliment to a JSP tag and JSF component
  • JSON - JavaScript Object Notation (i.e., object literals) defined using braces ({}) that contain a set of comma separated key value pairs
  • JSP tag - The JSF UIComponentELTag or UIComponentELBodyTag compliment to a JSF component and JSF renderer
  • Subcomponent - A child component used to render a complex component
  • Widget - Entity consisting of widget JavaScript and CSS, not necessarily based on the Dojo toolkit
  • Widget JavaScript - The JavaScript used to populate an HTML template and/or update component properties
  • Contents - JSON Array which contain JSON objects of sub components to be displayed inside the widget
  • Widget renderer - The JSF renderer used to output in-line JavaScript for instantiating a Dojo Widget

3.0 Component Usage

Woodstock components can be used in a few different ways. The most typical way to use a component is with the JSP tag. For example, to include an alert component in the JSP page, the web app developer will include a tag like this.

<webuijsf:alert id="msg1" type="information" summary="Patch Installed Successfully" detail="Patch 9997-01 was successfully installed on host alpha, beta and zed." />

Using the JSP tag, all the in-line JavaScript necessary to generate a Dojo widget is automatically output in the rendered HTML page. This includes the name of the desired widget, any localized and themed properties required to populate the HTML template, and any JavaScript files (e.g., widget JavaScript) necessary to support the Dojo Widget. However, a Dojo Widget can also be represented by the following JavaScript. This JavaScript makes the Woodstock component available to web app developers who cannot, or do not want to, use the JSP and/or JSF frameworks.

Note that any one of the key-value pairs of properties below can contain more key-value pairs, possibly used by subcomponents.

dojo.require("webui.<theme>.widget.alert");
webui.<theme>.widget.common.createWidget({
    _widgetType: "webui.<theme>:alert",
    templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/alert.html",
    summary "Patch Installed Successfully",
    id: "form1:msg1",
    detail: "",
    type:"information"
});

When the component is rendered, a DOM object corresponding to the component is created. To manipulate the component on the client side, you may invoke functions on the DOM object. With reference to the DOM id, to disable the component, invoke document.getElementById(id).setProps({detail: "New Detail Text"}).

4.7 Component Extensibility

In order to support developer defined type a set of Indicator is supported by Alert component.
An Indicator supports type, sort and image attribute. A String called "type" identifies an Indicator. It is the same as the value given to the Alarm "severity" property or to the Alert "type" property.
Each indicator has a sort value. Even Alert indicators can have sort values.

An example showing the use of developer define types:

<webuijsf:alert id="alert1" indicators="#{AlertBean.customIndicator}" type="myType" summary="Summary Text" />

Backing Bean:

 AlertBackingBean {
 ...
 public List getCustomIndicator() {
        
        FacesContext context = FacesContext.getCurrentInstance();
        UIComponent comp = context.getViewRoot().findComponent("form1:alert1");
        Alert alert = (Alert) comp;
               
        Indicator indicator = new Indicator("ALARM_CRITICAL_SMALL", "myType", 600);
        
        List lst = Alert.getDefaultIndicators();
        lst.add(indicator);
                
        return lst;
    }  
    ...
 }

8.0 Component High level design

8.1 Client side widget

The client side alert widget makes use of the fillInTemplate function to generate a template based widget. The alert widget first registers the setProps function on its widget in the fillInTemplate function and then makes a call to setProps function which sets the properties common to the alert widget. A getProps() is also registered in the fillInTemplate which is used to retrieve attribute values which setProps() function uses.

The alert widget supports custom or developer defined indicators. A json indicators object which contains type and image widget
is used to output set of indicators which includes default types in addition to developer defined type if specified.

Alert widget outputs complete set of indicators which enables client side update of indicators.

8.2 JSF Renderer

Alert component uses a set of Indicator objects that allows to  define the custom indicators including default indicators. Also
the sortValue property of Indicator object is used to sort indicators.

Alert renderer out puts all the required properties which is used by alert widget to render component.

8.3 JSF Renderer (Ajax)


Alert renderer (ajax) handles ashyncronous requests. This enables asynchronously refreshing the component
