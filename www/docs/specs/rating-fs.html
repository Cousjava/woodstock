1.2 Component Description

The Rating component displays a series of images, each of which signifies a rating (or grade) level.  There are two sets of interfaces in this component: A client side javascript interface that allows user to dynamically manipulate the widget and a server side JSF based component interface.

1.3 Component Purpose

The Rating component is intended to be used in cases where it desireable for the user to be able to assign a rating to an item.  The user need only click on one of a series of images to assign a rating.  This component was originally made available as part of Blueprints project (now obsolete), but which has been requested to be integrated into the Woodstock component library and made available in the VisualWeb interface of Netbeans.

There are some client-side behavioral characteristics which are important to highlight here. Hovering over any image BUT the mode toggle image (the "people" image) shows a preview of what you'll get if you eventually click on it.  For example, if the current grade is 4, then the 1st 4 stars are color-filled.  If you hover over star #3, then the 1st 3 stars will be color-filled.  When you move the mouse off that star #3, then the preview goes away and you see 4 color-filled stars again.  Hovering over the mode toggle image shows a preview of the other mode.

2.1.2 Functional requirements

Although no formal requirements have been provided, we can extract the following requirements based on the Blueprint rating component:

2.1.2.1 Must support a configureable number of grades (stars).
2.1.2.2 Must optionally allow the user to be able to indicate no interest in the rating.
2.1.2.3 Must optionally allow the user to clear his/her grade.
2.1.2.4 Must optionally allow switching the component to show the average grade of several user grade settings.
2.1.2.5 Must optionally allow specification of custom hover text to be rendered upon mouseover on each image.
2.1.2.6 Must optionally allow specification of acknowledgement hover text to be displayed upon mouseclick on each image.
2.1.2.7 Must be able to render read-only, so that the user cannot select a grade or view the average grade.
2.1.2.8 Must allow the developer to register Javascript callback functions when any image is clicked.

2.1.3 Performance requirements

This component is a highly interactive client-side component and therefore must not negatively impact normal user interaction with the mouse.

3.0 Component Usage

The most typical way to use a Woodstock component is with a JSP tag.  For example, the Rating component usage in a JSP page might look like this:

  <webuijsf:rating id="rating" 
      autoSubmit="true"
      maxGrade="#{RatingBean.maxGrade}"
      includeNotInterested="true" 
      includeClear="true"
      gradeHoverTexts="#{RatingBean.gradeHoverTexts}"
      notInterestedHoverText="#{RatingBean.notInterestedHoverText}"
      clearHoverText="#{RatingBean.clearHoverText}"
      gradeAcknowledgedText="#{RatingBean.gradeAcknowledgedText}"
      averageGrade="#{RatingBean.averageGrade}"
      grade="#{RatingBean.grade}" />
where the attribute values are defined in the RatingBean as follows:
 import com.sun.webui.jsf.component.Rating;

  public class RatingBean {
      private int maxGrade = 5;
      private int grade = Rating.CLEAR_GRADE; // initially clear
      private double averageGrade = 0.0;
      private int gradeSum = 0; // sum of all grade selections
      private int numGrades = 0; // total number of grades

      private String[] gradeHoverTexts = {
          "Hate it",
          "Below Average",
          "Average",
          "Above Average",
          "Love it"
      };

      public int getMaxGrade() {
          return maxGrade;
      }

      public String[] getGradeHoverTexts() {
          return gradeHoverTexts;
      }

      public String getNotInterestedHoverText() {
          return "Not Interested";
      }

      public String getClearHoverText() {
          return "Clear Rating";
      }

      public String getGradeAcknowledgedText() {
          return "Thank you!";
      }

      public int getGrade() {
          return grade;
      }

      public void setGrade(int grade) {
          this.grade = grade;
 
          // Not interested nor clear grade factors into average grade.
          if (this.grade == Rating.NOT_INTERESTED_GRADE) || this.grade == Rating.CLEAR_GRADE) {
              return;
          } 
 
          gradeSum += grade;
          numGrades++;
      }

      public double getAverageGrade() {
          if (numGrades == 0)
              return 0.0;
 
          // Compute average grade, rounded to neaest .001
          averageGrade = (double)gradeSum / numGrades;
          averageGrade = Math.rint(averageGrade * 1000) / 1000;
          return averageGrade;
      }
  }
Using the JSP tag, all the in-line JavaScript necessary to generate a Dojo widget is automatically output in the rendered HTML page. This includes the name of the desired widget, any localized and themed properties required to populate the HTML template, and any JavaScript files (e.g., widget JavaScript) necessary to support the Dojo Widget.  So the above example would yield:
  webui.suntheme.widget.common._createWidget("spanID",{
   "id": "form:rating",
   "autoSubmit" : true,
   "maxGrade": 5,
   "grade": 0,
   "averageGrade": 0.0,
   "widgetType": "webui.suntheme.widget.rating",
   "includeNotInterested": true,
   "includeClear": true,
   "gradeHoverTexts": [
     "Hate It",
     "Below Average",
     "Average",
     "Above Average",
     "Love It"
   ],
   "notInterestedHoverText": "Not Interested",
   "clearHoverText": "Clear Rating",
   "gradeAcknowledgedText": "Thank you!"
  });
This JavaScript can also be coded directly by web app developers who cannot, or do not want to, use the JSP and/or JSF frameworks.

7.1 Accessibility (A11Y)

Accessibility support will be provided in the following ways.  The widget will function as the ARIA role "radiogroup".
  • Alternate descriptions (via alt, title,  or longdesc attribute) will be provided for all images.
  • Keyboard focus can be added to tab navigation via tabindex attribute.
  • Once the widget receives focus, left/right arrows can be used to navigate to the previous/next image.
  • Entering the right arrow key when focus is on the last image will set focus to the first image.
  • Entering the left arrow key when focus is on the first image will set focus to the last image.
  • Tabbing into the widget will set focus to the image representing the grade.  If there is no grade (or the grade had been cleared), then focus is set to the first (if Tabbing) or last (if Shift-Tabbing) image.
  • Entering spacebar on a grade image selects that grade.  Entering spacebar on the mode toggle image toggles the display mode.
With respect to widget behavior, when navigating to each star via the arrow keys, each image that receives focus acts the same as when hovering - show a preview of what you'd get if you eventually hit the spacebar.  This presents a problem in that while navigating thru the widget with the arrows, you can never see what the current state actually is unless you tab out of the widget.  This is where it differs from a radiobutton group.   Therefore Control+Arrow can be used to move focus from one grade to the next without showing a preview or selecting a grade.
