1.2 Component Description

The imageHyperlink JSF component now renders itself with the help of a client side renderer instead of an html renderer. This means that there will exist a client side imageHyperlink widget for rendering the imageHyperlink component

1.3 Component Purpose

The ImageHyperlink component is a specialized hyperlink component that renders an image along with the regular hyperlink. The ImageHyperlink component allows you to specify different images to be shown when the widget in the enabled state and disabled state.Also, the imageHyperlink widget allows you to specify an url to navigate to as well submit the form when you click on the hyperlink so that the JSF navigation mechanism can be followed.

3.0 Component Usage

The  imageHyperlink tag can be used in a JSP page the following way. there exists an "disabledImage" facet which allows you to specify an image that should be shown when the hyperlink should be disabled. If this facet does not exist, then the same image is shown when the imageHyperlink is in enabled or the disabled state.

<webuijsf:imageHyperlink id="imhyperlink" text="Sun Microsystems" url="http://sun.com" imageUrl="/hyperlink/myyahoo.gif" target="_blank">
    <f:facet name="disabledImage">
       <webuijsf:image id="image2" url="/hyperlink/sun.gif"/>
    </f:facet>
</webuijsf:imageHyperlink>

This will output the necessary javascript needed to create the anchor widget on the client side. The output would look something like this.

 
webui.suntheme.widget.common.createWidget({
    "formId": "form1",
    "_widgetType": "webui.<theme>:imageHyperlink",

    "href": "http://sun.com",
    "disabled": false,
    "target":"_blank",
    "imagePosition": "right",
    "enabledImage": {
        "templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/image.html",
        "_widgetType": "webui.<theme>:image",
        "width": 13,
        "height": 13,
        "visible": true,
        "src": "/<appname>/faces/hyperlink/myyahoo.gif",

        "&nbsp;"
    ],

        "templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/image.html",

The imageHyperlink widget can be updated in the following way:
    • Get an handle to to the imageHyperlink dom node
      var domNode =  document.getElementById("form1:imhyperlink")
    • Use the getProps function that is available on the dom node to obtain all the properties that are set on that particular dom element.
      •  var props = domNode.getProps(); // returns a json object.
    • Use the setProps function to specify the properties to be updated for the particular component as a json object.
       domNode.setProps({disabled:true});
Note: <appname> is replaced by the application name.
         <theme> tag is replaced by the theme name.

8.0 Component High level design

8.1 Client side Widget:

The imageHyperlink widget supports two different states called the enabled and disabled states. In the enabled state the imageHyperlink can be clicked and you can navigate to a different page. In the disabled state the imageHyperlink will appear as plain text. Clicking on the imageHyperlink doesnt generate any form of action in the disabled state.

The imageHyperlink widget also supports two forms of page navigations.
  • Use the href attribute to navigate to a particular url.
  • Submit the form when the imageHyperlink is clicked
The client side widget also supports two different kinds of contents - "enabledImage" and "disabledImage". These are used if the image of the  imageHyperlink widget is to be changed depending on the enabled or the disabled state of the widget.
The imageHyperlink widget  decides whether or not to submit the page or use the href attribute to navigate depending on whether the href attribute is specified or not for the widget. If no href attribute is specified for the dom element will have a default value of "#" set. This is to get link look and feel.

8.2 JSF Component Renderer:

The ImageHyperlink widget renderer inherits from the Hyperlink's widget renderer. The contents JSON property in this case contains the children of the ImageHyperlink component and the text of the imageHyperlink. The image to be shown for the enabled state of the imageHyperlink component goes into the "enabledImage" JSON property. If the disabledImage facet is specified, the image specified in that facet goes inside the disabledImage JSON property.
