1.2 Component Description

The radioButtonGroup component is a group of one or more individual radio button elements with the same name.
Only one radiobutton can be selected in a group at any given time.

1.3 Component Purpose

The component still maintains its original purpose However the new client side rendering will enable updating radioButtonGroup component client side.

The radioButtonGroup widget can be updated in the following way:
  • Get an handle to to the radioButtonGroup dom node. Note that the component id must be used here for getting the handle on the dom element rather than the client id.
    var domNode =  document.getElementById("form1:rbGroup1")
  • Use the getProps function that is available on the dom node to obtain all the properties that are set on that particular dom element.
    •  var props = domNode.getProps(); // returns a json object.
  • Use the setProps function to specify the properties to be updated for the particular component as a json object.
     domNode.setProps({disabled: true});

3.1 Updating the component Asynchronously

The radioButtonGroup component also supports asynchronous updates. For example you could update the label of the component asynchronously the following way.
This example uses a text field whose value is bound to a backing bean. The radioButtonGroup component's label value is also bound to the same backing bean.
When an user enters text in the text field, the corresponding text will be updated in the radioButtonGroup.

Here is how the code looks like.

               <webuijsf:radioButtonGroup id="rb3" items="#{rbcbGrp.array}"
                                     selected="#{rbcbGrp.rb1selected}"
                                      label="#{RbCbBean.label}"
                                      toolTip="#{RbCbBean.label}" />                                                                                                           
               <br /><br />   
               <webuijsf:textField id="textField1" label="Enter new label" onKeyUp="refreshGroup('form1:rb3', 'form1:dynamicLabel2')" text="#{RbCbBean.label}"/>
               <br /><br />           

Note that the textField's onKeyUp attribute refers the refreshGroup javascript function.
This function will invoke the "refresh" function that is located on the html radioButtonGroup element's dom.
The argument that is passed to the refresh function would be the client id of the text field.
Hence the javascript function would look something like this.

                <script type="text/javascript">
                    function refreshGroup() {
                        // Get text to refresh.
                        var domNode = document.getElementById("form1:rb3");
                        domNode.refresh("form1:textField1"); // Update text field value and refresh.
                    }
               </script>

By default, the radioButtonGroup component's ajax functionality is set to true. i.e. the ajaxify attribute of the radioButtonGroup component returns true by default.
You can turn off the ajax functionality by setting the ajaxify attribute to false while creating the radioButtonGroup component.

                <webuijsf:radioButtonGroup id="rbgrp1"  ajaxify="false" items="#{rbcbGrp.array}"
                                     selected="#{rbcbGrp.rb1selected}"
                                      label="#{RbCbBean.label}"  />

The above created radioButtonGroup component will not exhibit any ajax functionality.

8.0 Component High level design

8.1 Client side Widget:

The client side radioButtonGroup widget makes use of the fillInTemplate function to generate a template based widget. The radioButtonGroup widget first registers the setProps function on its widget in the fillInTemplate function and then makes a call to setProps function which sets the properties common to the radioButtonGroup widget. A getProps() is also registered in the fillInTemplate which is used to retrieve attribute values which setProps() function uses.

8.2 JSF Component Renderer:

radioButtonGroup renderer out puts all the required properties which is used by radioButtonGroup widget to render component.
