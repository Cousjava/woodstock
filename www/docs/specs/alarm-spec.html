  • HTML template - The file used to define HTML tags associated with a component
  • In-line JavaScript - JavaScript used to instantiate a Dojo Widget, located anywhere (i.e., in-line) in the rendered HTML page
  • JSF - JavaServer Faces
  • JSF component - The JSF UIComponent compliment to a JSP tag and JSF renderer
  • JSF facet - A facet is a subcomponent that has a specific purpose defined by its parent container
  • JSF renderer - The JSF Renderer compliment to a JSP tag and JSF component
  • JSON - JavaScript Object Notation (i.e., object literals) defined using braces ({}) that contain a set of comma separated key value pairs
  • JSP tag - The JSF UIComponentELTag or UIComponentELBodyTag compliment to a JSF component and JSF renderer

  • Widget JavaScript - The JavaScript used to populate an HTML template and/or update component properties
  • Contents - JSON Array which contain JSON objects of sub components to be displayed inside the widget
  • Widget renderer - The JSF renderer used to output in-line JavaScript for instantiating a Dojo Widget

3.0 Component Usage

Woodstock components can be used in a few different ways. The most typical way to use a component is with the JSP tag. For example, to include an alarm component in the JSP page, the web app developer will include a tag like this.

<webuijsf:alarm id="alarm1" severity="major" text="major severity" />

Using the JSP tag, all the in-line JavaScript necessary to generate a Dojo widget is automatically output in the rendered HTML page. This includes the name of the desired widget, any localized and themed properties required to populate the HTML template, and any JavaScript files (e.g., widget JavaScript) necessary to support the Dojo Widget. However, a Dojo Widget can also be represented by the following JavaScript. This JavaScript makes the Woodstock component available to web app developers who cannot, or do not want to, use the JSP and/or JSF frameworks.

Note that any one of the key-value pairs of properties below can contain more key-value pairs, possibly used by subcomponents.

dojo.require("webui.<theme>.widget.alarm");
webui.<theme>.widget.common.createWidget({
    _widgetType: "webui.<theme>:alarm",
    templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/alarm.html",
    text "major alarm",
    id: "form1:alarm1",
    severity:"major"
});

When the component is rendered, a DOM object corresponding to the component is created. To manipulate the component on the client side, you may invoke functions on the DOM object. With reference to the DOM id, to change the text, invoke document.getElementById(id).setProps({text: "New Text"}).

4.7 Component Extensibility

In order to support developer defined severity a set of Indicator is supported by Alarm component.
An Indicator supports type, sort and image attribute. A String called "type" identifies an Indicator. It is the same as the value given to the Alarm "severity" property or to the Alarm "type" property.
Each indicator has a sort value.

An example showing the use of developer define types:

<webuijsf:alarm id="alarm11" indicators="#{AlarmBean.customIndicator}" severity="mySeverity" text="custom severity" />

Backing Bean:

 AlarmBackingBean {
 ...
 public List getCustomIndicator() {
        
        FacesContext context = FacesContext.getCurrentInstance();
        UIComponent comp = context.getViewRoot().findComponent("form1:alarm1");
        Alarm alert = (Alarm) comp;
               
        Indicator indicator = new Indicator("ALARM_CRITICAL_SMALL", "mySeverity", 600);
        
        List lst = Alarm.getDefaultIndicators();
        lst.add(indicator);
                
        return lst;
    }  
    ...
 }

8.0 Component High level design

8.1 Client side widget

The client side alarm widget makes use of the fillInTemplate function to generate a template based widget. The alarm widget first registers the setProps function on its widget in the fillInTemplate function and then makes a call to setProps function which sets the properties common to the alarm widget. A getProps() is also registered in the fillInTemplate which is used to retrieve attribute values which setProps() function uses.

The alarm widget supports custom or developer defined indicators. A json indicators object which contains type and image widget
is used to output set of indicators which includes default severities in addition to developer defined severity if specified.

Url, if specified will replace the image for all indicator and same is also true for height and width attribute. Rest of the supported image attributes will only be applicable to specified severity image.

Alarm widget outputs complete set of indicators which enables client side update of indicators.

8.2 JSF Renderer

Alarm component uses a set of Indicator objects that allows to  define the custom indicators including default indicators. Also
the sortValue property of Indicator object is used to sort indicators.

Alarm renderer out puts all the required properties which is used by alert widget to render component.

8.3 JSF Renderer (Ajax)


Alarm renderer (ajax) handles ashyncronous requests. This enables asynchronously refreshing the component
