1.2 Component Description

The anchor component behaves as an html anchor element.

1.3 Component Purpose

The anchor component  provides all the functionalities of an html anchor element. The existing anchor componentonly has attributes which enable the anchor component to anchor a particular location in the page. Hence, to make the anchor component provide the functionalities of an html anchor element, new attributes have been added which enable the component user to specify attributes supported by the html anchor element.

3.0 Component Usage

The anchor tag can be used in a JSP page the following way:

<webuijsf:anchor id="anchor1" text="Sun Microsystems" url="http://sun.com" target="_blank"/>

This will output the necessary javascript needed to create the anchor widget on the client side. The output would look something like this.

webui.<theme>.widget.common.createWidget({
    _widgetType: "webui.<theme>:anchor",
    templatePath": "/<appname>/theme/com/sun/webui/jsf/<theme>/templates/anchor.html",
    id: "form1:anchor1",
    name: "form1",
    href: "http://sun.com",
    target: "_blank"
    contents:[{"Sun Microsystems"}],
});

The anchor widget can be updated in the following way:
  • Get an handle to to the anchor dom node. Note that the component id must be used here for getting the handle on the dom element rather than the client id.
    var domNode =  document.getElementById("anchor1")
  • Use the getProps function that is available on the dom node to obtain all the properties that are set on that particular dom element.
    •  var props = domNode.getProps(); // returns a json object.
  • Use the setProps function to specify the properties to be updated for the particular component as a json object.
     domNode.setProps({href:"http://www.sun.com"});

Note: <appname> is replaced by the application name.
         <theme> tag is replaced by the theme name.

3.1 Updating the component Asynchronously

The anchor component also supports asynchronous updates. For example you could update the text of the component asynchronously the following way.
This example uses a text field whose value is bound to a backing bean. The anchor component's text value is also bound to the same backing bean.
When an user enters text in the text field, the corresponding text will be updated in the anchor.

Here is how the code looks like.

                    <webuijsf:anchor id="anchor7"  text="#{HyperlinkBean.dynamicText}" url="http://google.com"/>
                    <br/><br/>                           
                    <webuijsf:textField id="textField1" onKeyUp="refreshAnchor();" text="#{HyperlinkBean.dynamicText}"
                      label="Enter new text"/><br/>  

Note that the textField's onKeyUp attribute refers the refreshAnchor javascript function.
This function will invoke the "refresh" function that is located on the html anchor element's dom.
The argument that is passed to the refresh function would be the client id of the text field.
Hence the javascript function would look something like this.

                <script type="text/javascript">
                    function refreshAnchor() {
                        // Get text to refresh.
                        var domNode = document.getElementById("anchor7");
                        domNode.refresh("form1:textField1"); // Update text field value and refresh.
                    }
               </script>

By default, the anchor component's ajax functionality is set to true. i.e. the ajaxify attribute of the anchor component returns true by default.
You can turn off the ajax functionality by setting the ajaxify attribute to false while creating the anchor component.

                <webuijsf:anchor id="anchor7"  ajaxify="false" text="#{HyperlinkBean.dynamicText}" url="http://google.com"/>

The above created anchor component will not exhibit any ajax functionality.

8.0 Component High level design

8.1 Client side Widget:

The client side widget javascript abstracts out the properties to be set on the anchor element to a separate function called setAnchorProps.If any other widget intends to emulate anchor like properties, then it can invoke this function on its widget node for setting the anchor properties. Also note that, if a name property exists for the anchor widget, the same value is applied to the id attribute of the dom element. This is to ensure that when other elements such as the anchor or the hyperlink, refer to an anchor element, the particular anchor element needs to have both the id and the name attribute set to the same value for the anchor to be identified properly on the web page.

8.2 JSF Component Renderer:

The com.sun.webui.jsf.renderkit. html.AnchorRenderer extends from the javax.faces.renderer.Renderer class. It is not restricted to rendering instances of "com.sun.webui.jsf.component.Anchor". Properties are obtained via the UIComponent interface "getAttributes().get()". This renderer can be used for any UIComponent that contains the set or a sifficient subset of properties of an HTML "a" element

The anchor renderer also processes UIParameter components that are given as children to the anchor component and appends them as query parameters to the url that is specified. The name and value specified Note that the renderer does not html escape any characters for the UIParameter values and it is up to the developer to do so.The anchor component does not support any form of
JSF page navigation mechanism.

The anchor component does not support a "name" attribute  unlike the anchor widget. The AnchorRenderer assigns the component id of the anchor component as the value of the name attribute for the rendered widget. This follows existing anchor functionality and aids in avoiding complications that will happen if both the name and id attribute existed for the component.
